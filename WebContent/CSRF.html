  1. What is it:An authenticated user is tricked to issue unauthorized commands to the server who trusts this user
  2. Using this trick, attacker can issue a command to change pwd, email, or get access to whole website
  3. CSRF Prevention:
    1. CSRF Token/HMAC should be validated in each requests<
    2. The SameSite attribute can be used to control whether and how cookies are submitted in cross-site requests.
    3. Frequent ReAuth, checks like Captcha
    4. HTML attributes that are risky: GET, href, src
